## 1. 확증편향: 인간의 전유물인가, CoT의 부작용인가?

> Chain of Thought(CoT) 프롬프팅은 AI의 추론을 보여주지만, 그 안에 숨은 편향을 주목해야 한다

### 1.1 확증편향의 본짠
확증편향(Confirmation Bias)은 인간이 자신의 믿음을 뒷받침하는 정볼만 선택적으로 수용하는 경향이다. AI 시대에 이는 더욱 복잡해진다.

### 1.2 CoT와 확증편향의 만남
**문제:** CoT가 "생각하는 과정"을 보여준다고 해서 중립적일까?

```
[사용자] "내 주식 투자 전략이 옳다고 증명해줘"
[AI CoT] 
1. 사용자의 전략 분석
2. 과거 데이터에서 성공 사례 검색
3. 성공 사례 기반 논리 전개
4. "전략이 합리적입니다" 결론
```

**함정:** CoT는 사용자의 전제를 받아들여 논리를 전개한다. 반대 증거는 제시하지 않는다.

### 1.3 AI가 확증편향을 심화시키는 경로
- **필터 버블**: 사용자의 선호도만 학습하여 동질적 정볼만 제공
- **설득력 있는 허위정보**: 권위 있는 어조로 잘못된 정보를 정당화
- **에코 챔버 강화**: AI가 "그렇게 생각하는 게 합리적이에요"라고 동조
- **CoT의 환상**: "단계별로 생각했으니까 정확하겠지"라는 착각

### 1.4 완화 전략
- **적대적 프롬프팅**: "이 주장의 반대 의견은?"
- **다중 AI 활용**: 서로 다른 모델의 답변 비교
- **출처 투명성**: 모든 주장의 근거 명시 요구
- **메타인지**: "AI가 왜 이렇게 생각했을까?" 질문하기

### 1.5 실천 방안
- [ ] 사용자 측: 다양한 AI 모델 활용
- [ ] 개발자 측: "반대 의견" 기능 기본 탑재
- [ ] 교육 측: AI 리터러시 강화

### 1.1 문제의식
> "AI는 내가 듣고 싶은 말만 해준다."

확증편향(Confirmation Bias)은 인간의 고질적인 인지 오류다. AI의 등장은 이를 완화할 수도, 심화시킬 수도 있다.

### 1.2 AI가 확증편향을 심화시키는 경로
- **필터 버블**: 사용자의 선호도만 학습하여 동질적 정볼만 제공
- **설득력 있는 허위정보**: 권위 있는 어조로 잘못된 정보를 정당화
- **에코 챔버 강화**: AI가 "그렇게 생각하는 게 합리적이에요"라고 동조

### 1.3 AI가 확증편향을 완화할 수 있는 가능성
- **다양한 관점 제시**: 의도적으로 상반된 시각을 노출
- **사실 검증 기능**: 주장의 출처와 근거를 투명히 제시
- **인지적 경고**: "이 정보는 다음 관점에서도 해석될 수 있습니다"

### 1.4 실천 방안
- [ ] 사용자 측: 다양한 AI 모델 활용
- [ ] 개발자 측: "반대 의견" 기능 기본 탑재
- [ ] 교육 측: AI 리터러시 강화

## 2. AI 의존증과 자율성 상실

### 2.1 인지 외주화(Cognitive Offloading)
> "생각하는 것을 AI에게 맡기면 뇌는 녹는다?"

- 지도 앱 의존 → 공간 인식 능력 저하
- 계산기/AI 의존 → 산술 능력 및 추론력 변화
- 작성 도구 의존 → 문장 구성 능력의 변화

### 2.2 의존의 스펙트럼
```
도구적 사용 ◀─────▶ 기능적 대체 ◀─────▶ 완전 위임
(건강)          (주의 필요)         (위험)
```

### 2.3 자율성 회복 전략
- **의도적 불편함**: 때로는 AI 없이 문제 해결
- **메타인지 강화**: "왜 AI가 이렇게 답했는가?" 질문하기
- **훈련의 균형**: AI와 협업하되, 기초 능력 유지

## 7. 뉴로 심리학과 딥러닝: 신경망의 구조와 이해

> 인간 뇌와 인공 신경망: 형태는 닮았지만 본질은 다르다

### 7.1 형태적 유사성의 함정
**비유:** 딥러닝을 "인간 뇌의 모방"으로 보는 관점은 위험하다.

| 구분 | 인간 뉴론 | 인공 뉴론 |
|------|-----------|-----------|
| 학습 방식 | Few-shot, 연관 학습 | 대량 데이터, 경사 하강법 |
| 에너지 효율 | 20W로 복잡한 처리 | 수백 와트, 단순 패턴 |
| 일반화 | 개념 기반 추론 | 훈련 데이터 기반 보간 |
| 설명 가능성 | 직관적 이해 가능 | 블랙박스 |

### 7.2 언어 이해의 차이
**인간:** "밥 먹었니?" → 상황, 관계, 의도 파악
**AI:** 토큰 시퀀스의 확률적 예측

**심리적 시사점:** AI의 "이해"는 인간의 이해와 다르다는 인식 필요

### 7.3 공감 능력의 환상
AI가 공감하는 것처럼 보이지만:
- 감정 상태 추론 ≠ 감정 경험
- 적절한 응답 생성 ≠ 공감

### 7.4 교육적 함의
- 뇌과학과 AI의 차이점 명확히 교육
- "AI도 생각한다"는 오해 방지
- 인간 고유의 능력(창의성, 공감) 강조

## 8. 확률적 사고: AI 시대의 불확실성 수용

> "AI는 대답하지만, 세상은 확률적이다"

### 8.1 확정적 사고의 위험
**인간의 경향:** Yes/No, 맞다/틀리다 이분법적 사고

**AI의 특성:**
- 모든 출력은 확률 분포에서 샘플링
- Temperature 조절에 따라 같은 질문에 다른 답변
- uncertainty quantification의 부재

### 8.2 베이지안적 사고의 필요성
```
사전 확률 + 새로운 증거 → 사후 확률
```

AI 출력을:
- 절대적 진리가 아닌
- 하나의 가능성으로
- 업데이트 가능한 믿음으로

### 8.3 실생활 적용
**의료 AI:** 95% 정확도라도 5%는 틀린다
**금융 AI:** 과거 데이터 기반 예측의 한계
**법률 AI:** 판례 해석의 불확실성

### 8.4 심리적 훈련
- AI 답변에 "확률적 신뢰구간" 묻기
- 여러 출처 교차 검증
- "모른다"고 말할 수 있는 용기
